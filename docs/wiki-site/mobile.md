# Mobile App

`apps/mobile` is the Expo-based mobile client. Android is the primary first-beta surface. iPhone uses the same codebase but is not a first-beta commitment.

## Current Capabilities

- Email magic-link bootstrap with explicit 18+ affirmation
- Invite-only account creation (beta invite or group invite)
- Pseudonymous profile setup
- Device bundle registration and contact-card resolution
- Ciphertext DM send/receive via the relay mailbox
- Relay-hosted group threads and attachment upload/download
- Local SQLite message cache and SecureStore for credentials
- Android-native FCM push token registration

## Running Locally

```bash
# Start Metro bundler (Android emulator or device required)
npm run dev:mobile

# or from inside the apps/mobile directory
cd apps/mobile
npx expo start
```

Point an Android emulator at the local relay URL (set in `.env` or app config) or connect to the production relay.

## Build

### Local development build (Android)

```bash
npm run build:android
```

This produces an APK and/or AAB under `apps/mobile/android/app/build/outputs/`.

### CI release build

The GitHub Actions workflow `.github/workflows/release-android.yml` produces a signed release APK and AAB when a version tag is pushed.

```bash
git tag v1.2.3
git push origin v1.2.3
```

Signing uses the keystore stored as the `ANDROID_KEYSTORE_BASE64` GitHub Actions secret. See the [Operator Playbook](./operator-playbook.md) for credential management.

## Google Play Deployment

`.github/workflows/deploy-play-store.yml` builds a signed AAB and uploads it to the Play Store when triggered. It requires the `GOOGLE_PLAY_SERVICE_ACCOUNT_JSON` secret to be configured in the repository.

## Environment Configuration

Copy the example env before building:

```bash
cp apps/mobile/.env.example apps/mobile/.env
```

Key variables:

| Variable                | Purpose                                                                               |
| ----------------------- | ------------------------------------------------------------------------------------- |
| `EXPO_PUBLIC_RELAY_URL` | Base URL of the relay (defaults to `http://10.0.2.2:8787` for local Android emulator) |
| `EXPO_PUBLIC_WEB_URL`   | Web app URL for invite landing links                                                  |

## FCM Setup

Android push notifications require a Firebase project and a `google-services.json`. See `docs/android-fcm-setup.md` in the repository for the full setup guide.

## iPhone (Deferred)

The iPhone scaffold lives in the same `apps/mobile` Expo project. A simulator build is generated by `.github/workflows/release-apple.yml`. Signed TestFlight and App Store distribution are deferred until the Android/Windows/Ubuntu first-wave surfaces are stable.

## Local SQLite Schema

The mobile client persists:

- Decrypted DM history (local-only, never synced to relay)
- Attachment metadata
- Contact card cache
- Pending envelope queue

All sensitive credentials are stored in Expo SecureStore, not SQLite.
